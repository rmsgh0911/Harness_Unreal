# Harness Work Layer

The work layer stores the current operating state for iterative agent work.

## Files

- `state.md`: latest confirmed project facts only.
- `next.md`: unresolved work, deferred risks, and human decisions needed.
- `cycles/`: short date-based work loop records.

Keep this layer compact. Do not use it as a project encyclopedia; put project structure and routing hints in `Harness/Claude/index/`.
