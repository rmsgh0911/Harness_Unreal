# Task Records

Create one file per task or branch: `Harness/work/tasks/<task-id>.md`.

Copy `task.example.md` and record the owner, branch, worktree, timestamps, status, scope, success criteria, and remaining work. Keep the task ID filesystem-safe and use the same ID with `harness_cycle.py --task`.

Use a branch-unique task ID when multiple worktrees may create similarly named work.
